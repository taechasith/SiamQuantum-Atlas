Local workflow: submissions persist immediately in SQLite, then move through a local queue. Automation is best-effort only and does not pretend to be fully hands-free.

Workflow states

This page is now a real local intake workflow. It records the submission immediately, then exposes queue state instead of pretending the downstream pipeline is always automatic.

  • queued: stored and accepted
  • processed: follow-up work completed
  • queued_no_source or queued_no_text: stored, but enrichment could not proceed
  • failed: request persisted, background follow-up failed

สถานะหลังส่ง / What happens next

เมื่อรับ submission แล้ว ระบบจะบันทึกแถวใน `community_submissions` และจัดคิวทำงานเบื้องหลังเท่าที่สภาพแวดล้อมรองรับ.

  • รับค่า `handle` และ `url` แล้วตั้งสถานะเริ่มต้นเป็น queued
  • พยายามเรียก `claude.extract_triplets` หากมี source text พร้อมใช้งาน
  • รีเฟรชเส้นทางวิเคราะห์ที่มีอยู่เพื่ออัปเดต stat state เท่าที่ทำได้
  • ล้างเฉพาะ cache กลุ่ม `ttest:` โดยไม่กระทบ cache อื่น
